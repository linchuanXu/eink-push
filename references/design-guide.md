# 阅星曈卡片设计规范

AI 写自由 HTML 时遵守本文档，确保在 480×800 墨水屏上正常渲染。

---

## 版式目标（卡片 vs 电子书）

**卡片截图**要当成**一页 PPT**、一块**仪表盘屏**或一张**资讯阅读卡**来排版：信息分**模块**呈现，扫一眼有重点；**不要**做成单栏长文网页（大段段落从头读到尾）。需要连续阅读的长文、多节报告 → 用 **Markdown → epub**（见 `SKILL.md` 分流），不要用 HTML 卡片冒充文章版式。

**推荐结构（按需组合，不必每次全套）：**

| 区域 | 作用 | 常见手法 |
|------|------|----------|
| 顶区 | 场景、时间、角标 | 左右对齐一行，或小标签 |
| 主标题区 | 一眼知道主题 | 大字号 + 可选副标题 |
| 中区 | 核心信息 | **栅格 / 多卡并列**（状态、来源、数字）、或 **列表条**（每条有边框或浅底） |
| 焦点区 | 一条最重要结论/通知 | 加粗框、浅底、或与周围留白对比 |
| 底区 | 收束 | 一句标语、进度提示、来源说明 |

**忌讳：** 整屏只有「标题 + 长段落 + bullet」一种层级；那是文档流，预览感弱，也不利于墨水屏「快扫」。应用边框、间距、分栏把层次拉开。

上表与 `references/framework-samples/` 里的页面**只是示例节奏**，不是必须照抄的分栏模版。

## 表现力（网页想画什么都可以）

在固定画布内，**任意合法 HTML/CSS** 都可以用来「构图」，只要仍满足字号、对比度、无外链图等硬约束。不必把每张卡都做成同一种「顶栏 + 三格 + 虚线框」。

**常用手段（可组合、可只用其中几种）：**

- **内联 SVG**：流程箭头、小图标、示意节点、简单图表；不依赖 `background-image: url(http…)`。
- **纯 CSS 造型**：圆角卡片、台阶式色块、粗竖条强调、虚线/双线与留白一起玩层次。
- **错落排版**：标题与插图左右分栏、大小字强烈对比、悬挂编号、杂志式分区（多块 `div` 各管一段视觉）。
- **符号与序号**：Unicode（※ → ◇ 等）、特大号步骤数字、一行「金句」单独成块。

**仍建议避免：** 见下文「布局注意事项」中的动画、细线、中灰大块底、复杂渐变等——与「能不能画得花」无关，是墨水屏渲染特性。

## 画布

- **设备尺寸**：480 × 800 px。`render_image.py` 强制 `viewport=480×800`，截图 clip 到 (0,0,480,800)
- **自动缩放**：`render_image.py` 在截图前会注入 JS 检测 `body.scrollHeight`，若超出 800px 则对 body 应用 `zoom` 自动缩小；`base.html` 模板里同样内置此脚本作为双保险。内容可以超出 800px，渲染时会等比缩小，字号不会被截断。
- **容量参考（实测）**：30px 正文、行高 1.65、上下各 40px 内边距 → 每张卡约容纳 **100 字**。200 字建议拆 2 张，300 字拆 3 张，宁多勿挤。超出时自动缩字，但缩得太厉害（如缩到 0.6× 以下）阅读体验差，仍建议主动分页。
- **正确写法**：body 用 `100vw / 100vh`，Playwright 视口是 480×800，所以 `100vw=480px / 100vh=800px`，天然精确。同时在浏览器里也能铺满窗口，不需要 DevTools 辅助
- **为什么不用固定 `480px`**：Water/Pico/MVP 等框架有自己的 `max-width` / `padding`，固定 px 容易被覆盖，造成内容区溢出裁切
- **覆盖框架样式**（必须放在框架 CSS 之后，用 `!important` 全覆盖）：

```css
html { margin: 0 !important; padding: 0 !important; }
body {
  width: 100vw !important; height: 100vh !important;
  max-width: none !important; min-height: 0 !important;
  overflow: hidden !important; margin: 0 !important; padding: 0 !important;
}
```

- 布局容器（`.shell` / `.page` 等）用 `width:100%; height:100%` 继承 body 大小，再用 `box-sizing:border-box` 确保 padding 不溢出

## 颜色

墨水屏只有灰阶，所有颜色最终被转成 4 级灰。写 HTML 时直接用黑白灰即可，不要用彩色（写了也能渲染，但颜色信息丢失）：

| 用途 | 建议值 |
|------|--------|
| 文字 / 主要内容 | `#000000`（纯黑，不要用 #111 或 #333） |
| 次要文字 / 标签 | `#222222`（深深灰） |
| 边框 / 分隔线 | `#555555`（中深灰） |
| 浅色背景 / 高亮区 | `#cccccc` |
| 页面背景 | `#ffffff` |

## 字体

- 优先使用本地字体（已内置 **MiSans** 和 **Noto Serif SC**），`render_image.py` 会自动注入
- 也可以用 Tailwind CDN / Google Fonts，Playwright 联网加载
- **最小字号 26px**，低于此在 e-ink 上偏小，阅读费力
- **正文 30–36px**（`base.html` 默认 32px），标题 40–52px
- 副标题 / 标签 / 元信息可用 24–28px，但不要再小
- 行高 1.6 以上，中文大字体也需要充足行距

```css
/* 推荐字体声明 */
font-family: 'MiSans', 'Noto Serif SC', serif;
```

## 布局注意事项

**可以用：**
- Flexbox / Grid — 完全支持
- `border`, `border-radius` — 支持
- `box-shadow` — 支持（灰度渲染，淡阴影效果不明显，加深一点）
- Tailwind CDN — 直接引入，Playwright 联网时正常工作

**避免：**
- CSS 动画 / transition（截图是静止画面，动画无意义）
- `background-image: url(http://...)` 外链图片（慢，可能失败）
- 渐变（`linear-gradient`）——墨水屏灰阶转换后效果差，改用纯色块
- 极细线（`1px` 有时在抖点后消失，建议 ≥ 2px）
- 大面积中灰（`#888888`）背景——抖点后变成噪点，用浅灰或白

## 框架菜单（随内容氛围选一个）

**先定版式**（是否分区、是否有卡片/栅格），再选框架；框架只解决写法，不替代「仪表盘 / 资讯卡」式信息架构。以下框架均通过 CDN 引入，Playwright 联网时自动加载。也可选「无框架」离线使用。

---

### 1. Tailwind CSS — 现代、灵活、高密度信息
适合：数据卡、战情板、进度报告、结构化内容

```html
<script src="https://cdn.tailwindcss.com"></script>
```

常用灰阶 class：`text-black` `text-gray-800` `text-gray-600` `bg-white` `bg-gray-50` `bg-gray-100` `bg-gray-200` `border-gray-400` `border-gray-600`

---

### 2. Water.css — 无 class，素雅，阅读感强
适合：单页摘要、短语录；若内容多、要分区，优先 **拆多张卡片** 或改 Tailwind / 无框架做栅格，避免一屏堆成长文感

```html
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/water.css@2/out/light.css">
```

只需写语义 HTML（`<h1>` `<p>` `<blockquote>` `<ul>`），自动好看。

---

### 3. Pico.css — 简洁优雅，衬线感
适合：简报、阅读型卡片、知识摘要

```html
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/@picocss/pico@2/css/pico.min.css">
```

同样写语义 HTML 即可，风格比 Water.css 更工整。

---

### 4. MVP.css — 极简主义，大字留白
适合：一句话金句、Today I Learned、单条重点

```html
<link rel="stylesheet" href="https://unpkg.com/mvp.css">
```

---

### 5. 无框架（离线，最快）
适合：需要完全掌控布局，或网络不可用时

直接在 `<style>` 里写 CSS，或从 `assets/templates/base.html` 读取起点（已含 reset、canvas、色板变量）。

## 质量检查（写完 HTML 后自问）

- [ ] **版式**是否像一屏 PPT / 海报 / 资讯卡（有分区或视觉锚点），而不是单栏长文？（不要求与样例同构。）
- [ ] 是否至少有一处**非纯文字流**的抓手（小图、SVG、大号数字、明显形态分区等）？
- [ ] 内容是否超出 800px 高度？（加 `overflow:hidden` 强制裁切）
- [ ] 正文字号 ≥ 30px，标题 ≥ 40px？（墨水屏宜大不宜小）
- [ ] 正文颜色是纯黑 `#000000`，次要文字用 `#222222`？
- [ ] 没有用纯彩色作为主要信息载体？（颜色会丢失）
- [ ] 没有依赖外链图片？
